# Client Work Management System (Full Stack)

This repository contains a complete internal work management system with:

- `client-work-backend` (Node.js + Express + MongoDB API)
- `frontend` (React + Vite + TypeScript UI)

## Repository Structure

```text
.
├── client-work-backend/
├── frontend/
└── README.md
```

## Tech Stack

### Backend
- Node.js + Express.js
- MongoDB + Mongoose
- JWT auth (access + refresh)
- Cookie-based refresh token
- Validation + rate limiting

### Frontend
- React 18 + Vite
- TypeScript
- Tailwind CSS + shadcn/ui
- TanStack Query
- React Hook Form + Zod

## Prerequisites

- Node.js 18+
- npm 9+
- MongoDB connection string

## 1) Backend Setup (`client-work-backend`)

```powershell
cd "client-work-backend"
npm install
```

Create environment file:

```powershell
copy .env.example .env
```

Set required values in `.env`:

- `PORT=5000`
- `MONGODB_URI=...`
- `JWT_ACCESS_SECRET=...`
- `JWT_REFRESH_SECRET=...`
- `CLIENT_URL=http://localhost:5173`

Optional seed admin values:

- `SEED_ADMIN_NAME`
- `SEED_ADMIN_EMAIL`
- `SEED_ADMIN_PASSWORD`

Seed admin (optional but recommended):

```powershell
npm run seed
```

Run backend:

```powershell
npm run dev
```

Backend base URL:

- `http://localhost:5000/api`

## 2) Frontend Setup (`frontend`)

```powershell
cd "frontend"
npm install
```

Create frontend env file `.env.local`:

```env
VITE_API_URL=http://localhost:5000/api
```

Run frontend:

```powershell
npm run dev
```

Frontend URL:

- `http://localhost:5173`

## 3) Run Both (two terminals)

Terminal 1:

```powershell
cd "client-work-backend"
npm run dev
```

Terminal 2:

```powershell
cd "frontend"
npm run dev
```

## Main Features

- Admin authentication
- Dashboard statistics
- Client CRUD (create, edit, delete, search)
- Task CRUD (status, priority, due date, client mapping)
- Responsive layout for desktop/tablet/mobile
