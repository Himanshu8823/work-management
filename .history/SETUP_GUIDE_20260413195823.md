## рџЏ— Architecture

The application follows **Clean Architecture principles**:

### Frontend (React + Vite)
```
Presentation Layer в†’ Components
                  в†“
State Management в†’ TanStack Query (useQueries hooks)
                  в†“
Service Layer в†’ API Service (axios + interceptors)
                в†“
Data Layer в†’ Types (TypeScript interfaces)
```

### Key Design Decisions

1. **Authentication**: JWT tokens stored in HttpOnly cookies for security
2. **State Management**: TanStack Query for server state + React hooks for UI state
3. **Validation**: Zod schemas for both forms and API responses
4. **Forms**: React Hook Form for efficient form management
5. **Styling**: Tailwind CSS with shadcn/ui components for consistency

## вљ™пёЏ Setup Instructions

### Prerequisites
- Node.js (v18+)
- npm or yarn
- Backend API running on `http://localhost:5000`

### Step 1: Install Dependencies

```bash
cd frontend
npm install
```

This installs:
- React & React DOM
- Vite (build tool)
- TypeScript
- TanStack Query
- React Hook Form + Zod
- Tailwind CSS + shadcn/ui
- Axios for API calls
- React Router for navigation

### Step 2: Configure Environment

Create `.env.local`:

```env
VITE_API_URL=http://localhost:5000/api
```

This tells the frontend where to find the backend API.

### Step 3: Verify Backend Connection

Ensure the backend is running:

```bash
cd ../client-work-backend
npm run dev
```

The backend should be available at `http://localhost:5000`

## рџ”Њ API Integration

### How It Works

1. **Request Interceptor**: Automatically adds JWT token to headers
2. **Response Interceptor**: Handles 401 errors by refreshing token
3. **Error Handling**: Centralized error responses
4. **CORS**: Configured to accept requests from frontend origin

### API Service Structure

```typescript
// Simple API calls
apiService.login(email, password)
apiService.getClients(filters)
apiService.createClient(data)
apiService.updateTask(id, data)

// Hook-based queries
useLogin()
useClients(filters)
useCreateClient()
useTasks(filters)
```

### Query Keys Strategy

Uses TanStack Query's query key factory pattern:

```typescript
queryKeys = {
  all: ['query'],
  clients: () => [...queryKeys.all, 'clients'],
  clientsList: (filters) => [...queryKeys.clients(), 'list', filters],
  client: (id) => [...queryKeys.clients(), id],
  tasks: () => [...queryKeys.all, 'tasks'],
  tasksList: (filters) => [...queryKeys.tasks(), 'list', filters],
  // ... etc
}
```

Benefits:
- Automatic cache invalidation
- Type-safe query management
- Efficient data fetching

## вњЁ Features Explained

### 1. Authentication

**Flow:**
```
Login в†’ Backend validates в†’ JWT token returned
        в†’ Token stored in HttpOnly cookie
        в†’ User redirected to dashboard
```

**Security:**
- Passwords hashed with bcrypt
- JWT tokens with expiration
- Refresh token mechanism
- HttpOnly cookies prevent XSS attacks

### 2. Dashboard

**Components:**
- Overview stat cards (clients, tasks, pending, completed)
- Recent tasks list
- Quick statistics

**Updates:** Cached for 2 minutes, automatic invalidation after mutations

### 3. Client Management

**CRUD Operations:**
- **Create**: Add new client with validation
- **Read**: List all clients with search
- **Update**: Edit client details
- **Delete**: Remove client (cascade deletes tasks)

**Features:**
- Real-time search
- Form validation with Zod
- Duplicate email prevention
- Optimistic updates

### 4. Task Management

**CRUD Operations:**
- **Create**: Link task to client, set status & priority
- **Read**: List with filters (status, client, search)
- **Update**: Edit task or change status
- **Delete**: Remove task

**Features:**
- Status workflow: Pending в†’ In Progress в†’ Completed
- Priority levels: Low, Medium, High
- Due date tracking
- Client association
- Bulk status updates
